
## 镜像名称

`soar_eval:latest`

远程仓库：`modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/sglang_test:2.0.3`

## 运行方式

```bash
docker run --gpus '"device=0"' \
  -v /path/to/model:/model/stage2_rc2_job_165691_iter_4400 \
  -v /path/to/gpqa_data:/data/gpqa \
  -v /path/to/eval_data_128.jsonl:/data/eval_data_128.jsonl \
  -e MODEL_PATH=/model/stage2_rc2_job_165691_iter_4400 \
  -e PORT=30000 \
  -e RECORD_ID=record_001 \
  -e USER_ID=user_001 \
  -e TASK_ID=task_001 \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/sglang_test:2.0.3
  
  #建议加上以下变量以便快速测试
  -e GPQA_MAX_SAMPLES=5
  -e BENCH_MAX_SAMPLES=10
```

## 环境变量

| 变量                  | 必填  | 说明                             | 默认值                                      |
| ------------------- | --- | ------------------------------ | ---------------------------------------- |
| `MODEL_PATH`        | 是   | 容器内模型路径                        | `/model/stage2_rc2_job_165691_iter_4400` |
| `PORT`              | 否   | SGLang 服务端口                    | `30000`                                  |
| `DATA_DIR`          | 否   | GPQA 数据集目录（需挂载）                | `/data/gpqa`                             |
| `EVAL_DATA`         | 否   | benchmark_duration 数据集路径（需挂载）  | `/data/eval_data_128.jsonl`              |
| `RECORD_ID`         | 是   | 提交记录ID                         | 空                                        |
| `USER_ID`           | 是   | 用户ID                           | 空                                        |
| `TASK_ID`           | 是   | 任务ID                           | 空                                        |
| `GPQA_MAX_SAMPLES`  | 否   | GPQA 每子集最多评测题数（调试用）            | 不传则跑全量                                   |
| `BENCH_MAX_SAMPLES` | 否   | benchmark_duration 最多数据条数（调试用） | 不传则跑全量 128 条                             |

## GPU 选择

通过 `--gpus` 参数控制：

- 全部 GPU：`--gpus all`
- 指定 GPU：`--gpus '"device=0"'`、`--gpus '"device=0,1"'`

## 输出格式

容器执行完毕后，**stdout 最后输出一段 JSON**：

```json
{
  "record_id": "test_001",
  "user_id": "test_user",
  "result": {
    "error_msg": "",
    "score": {
      "acc": 20.0,
      "benchmark_duration": {
        "S1": 262.91,
        "S8": 96.21,
        "Smax": 96.56
      }
    },
    "sort_by": "acc"
  },
  "state": "1",
  "task_id": "test_task"
}
```

### 字段说明

- `state`: `"1"` 表示成功，`"0"` 表示失败
- `error_msg`: 失败时包含具体错误信息，成功时为空字符串
- `acc`: GPQA diamond 子集的总准确率（百分制），失败时为 0
- `benchmark_duration`: 包含 3 个值，分别是 3 档并发度下跑完 128 条请求的墙钟时间（秒），越短越好
  - `S1`: 无并发（串行）
  - `S8`: 8 并发
  - `Smax`: 无并发上限（全部请求同时发送）
- `sort_by`: 排序字段，当前固定为 `"acc"`

## 容器内执行流程

1. 激活 SGLang-MiniCPM-UltraAttn 虚拟环境
2. 后台启动 SGLang 推理服务
3. 等待服务就绪（轮询 `/health`，最多等 10 分钟）
4. 运行 `gpqa_eval.py` 评测 GPQA diamond 子集，获取 `acc`
5. 运行 `bench_serving.sh`（基于 `sglang.bench_serving`），在 3 档并发度（1/8/无上限）下分别跑完 128 条请求，记录墙钟时间
6. 组装 JSON 结果输出到 stdout
7. 关闭 SGLang 服务，容器退出

## 预估耗时

GPQA_MAX_SAMPLES=5, BENCH_MAX_SAMPLES=10

- Benchmark 部分：S1（262.91s）+ S8（96.21s）+ Smax（96.56s）+ warmup 开销 ≈ 约 8 分钟
- GPQA 部分：5 条 diamond 题目，按之前 S1 单条 ~26s 估算，并发 8 大约 2-3 分钟
- SGLang 启动：加载模型大约 2-3 分钟，总计大约 12-15 分钟。
- 
对于全量评测（198 题 GPQA + 128 条 × 3 档 benchmark）的预估，按比例大约 4-6 小时。

## 后端调用示例（Python）

```python
import subprocess
import json

result = subprocess.run(
    ["docker", "run", "--gpus", '"device=0"',
     "-v", "/path/to/model:/model/stage2_rc2_job_165691_iter_4400",
     "-v", "/path/to/gpqa_data:/data/gpqa",
     "-v", "/path/to/eval_data_128.jsonl:/data/eval_data_128.jsonl",
     "-e", f"RECORD_ID={record_id}",
     "-e", f"USER_ID={user_id}",
     "-e", f"TASK_ID={task_id}",
     "modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/sglang_test:2.0.3"],
    capture_output=True, text=True, timeout=43200  # 12小时超时
)

# 从 stdout 提取最后的 JSON 块
lines = result.stdout.strip().split("\n")
json_start = None
for i, line in enumerate(lines):
    if line.strip() == "{":
        json_start = i
if json_start is not None:
    json_str = "\n".join(lines[json_start:])
    output = json.loads(json_str)
```
