# MiniCPM-SALA 模型推理服务

基于 [SGLang](https://github.com/sgl-project/sglang) 的 **MiniCPM-SALA** 高性能推理服务，提供 OpenAI 兼容 API 接口。

## 特性

- **OpenAI 兼容 API** — 支持 `/v1/chat/completions`、`/v1/completions` 等标准接口，可直接对接 OpenAI SDK
- **高性能推理** — 集成 `sgl-kernel`、`infllm_v2`（InfLLM v2 稀疏注意力）、`sparse_kernel` 等定制 CUDA 算子
- **MiniCPM 专属优化** — 默认使用 `minicpm_flashinfer` 注意力后端和 `dense-as-sparse` 策略
- **灵活部署** — 支持单卡/多卡 Tensor Parallel、环境变量一键配置、守护模式运行
- **Prometheus 监控** — 内置 `/metrics` 指标端点，便于接入可观测性平台
- **Docker 一键部署** — 预构建镜像开箱即用，也可从源码自行构建

## 快速开始

### 1. 拉取镜像

```bash
docker pull modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 2. 启动服务

```bash
docker run --rm --gpus '"device=0"' -p 30000:30000 \
  -v /path/to/MiniCPM-SALA:/models/MiniCPM-SALA \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

> 将 `/path/to/MiniCPM-SALA` 替换为宿主机上模型权重文件的实际路径。

### 3. 验证服务

```bash
# 健康检查
curl http://localhost:30000/health

# 查看已加载模型
curl http://localhost:30000/v1/models

# 对话补全
curl http://localhost:30000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "MiniCPM-SALA",
    "messages": [{"role": "user", "content": "你好"}],
    "max_tokens": 512
  }'
```

## 配置参数

所有参数均通过 `docker run -e` 环境变量传入，可按需覆盖：

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| `MODEL_PATH` | `/models/MiniCPM-SALA` | 容器内模型路径 |
| `PORT` | `30000` | 服务监听端口 |
| `HOST` | `0.0.0.0` | 服务监听地址 |
| `TP_SIZE` | `1` | Tensor Parallel 并行度 |
| `MAX_RUNNING_REQUESTS` | `32` | 最大并发请求数 |
| `ENABLE_METRICS` | `1` | 是否开启 Prometheus 指标（设为 `0` 关闭） |
| `SGLANG_SERVER_ARGS` | 见下方 | SGLang 启动参数（整体覆盖） |

默认 `SGLANG_SERVER_ARGS`：

```
--disable-radix-cache --attention-backend minicpm_flashinfer --chunked-prefill-size 8192 --skip-server-warmup --dense-as-sparse
```

> **注意：** 设置 `SGLANG_SERVER_ARGS` 会**完全覆盖**默认参数，请确保包含所有必要的 flag。

## 使用示例

### 指定 GPU 和端口

```bash
docker run --rm --gpus '"device=2"' -p 8080:8080 \
  -v /home/user/model/MiniCPM-SALA:/models/MiniCPM-SALA \
  -e PORT=8080 \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 自定义模型路径

如果模型不挂载到默认路径，可以通过 `MODEL_PATH` 指定：

```bash
docker run --rm --gpus '"device=0"' -p 30000:30000 \
  -v /data/my-custom-model:/workspace/model \
  -e MODEL_PATH=/workspace/model \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 多卡 Tensor Parallel

通过 `--gpus` 指定多卡，同时设置 `TP_SIZE`：

```bash
docker run --rm --gpus '"device=0,1"' -p 30000:30000 \
  -v /path/to/model:/models/MiniCPM-SALA \
  -e TP_SIZE=2 \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 调整并发数

```bash
docker run --rm --gpus '"device=0"' -p 30000:30000 \
  -v /path/to/MiniCPM-SALA:/models/MiniCPM-SALA \
  -e MAX_RUNNING_REQUESTS=64 \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 覆盖 SGLang 启动参数

```bash
docker run --rm --gpus '"device=0"' -p 30000:30000 \
  -v /path/to/MiniCPM-SALA:/models/MiniCPM-SALA \
  -e SGLANG_SERVER_ARGS="--disable-radix-cache --attention-backend minicpm_flashinfer --chunked-prefill-size 4096 --dense-as-sparse" \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 后台运行（守护模式）

```bash
docker run -d --name minicpm-sala \
  --gpus '"device=0"' -p 30000:30000 \
  --restart unless-stopped \
  -v /path/to/MiniCPM-SALA:/models/MiniCPM-SALA \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0

# 查看日志
docker logs -f minicpm-sala

# 停止服务
docker stop minicpm-sala
```

## API 接口

服务启动后提供 OpenAI 兼容的 API，主要接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/health` | GET | 健康检查（返回 200 表示就绪） |
| `/v1/models` | GET | 查看已加载模型列表 |
| `/v1/chat/completions` | POST | 对话补全（Chat） |
| `/v1/completions` | POST | 文本补全（Completion） |
| `/metrics` | GET | Prometheus 指标（需 `ENABLE_METRICS=1`） |

### 对话补全示例

```bash
curl http://localhost:30000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "MiniCPM-SALA",
    "messages": [
      {"role": "system", "content": "你是一个有帮助的AI助手。"},
      {"role": "user", "content": "请简要介绍一下量子计算。"}
    ],
    "max_tokens": 1024,
    "temperature": 0.7,
    "stream": false
  }'
```

### 流式输出

```bash
curl http://localhost:30000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "MiniCPM-SALA",
    "messages": [{"role": "user", "content": "写一首五言绝句"}],
    "max_tokens": 256,
    "stream": true
  }'
```

### Python 调用（OpenAI SDK）

```python
from openai import OpenAI

client = OpenAI(base_url="http://localhost:30000/v1", api_key="none")

response = client.chat.completions.create(
    model="MiniCPM-SALA",
    messages=[{"role": "user", "content": "你好"}],
    max_tokens=512,
)
print(response.choices[0].message.content)
```

### Python 流式调用

```python
from openai import OpenAI

client = OpenAI(base_url="http://localhost:30000/v1", api_key="none")

stream = client.chat.completions.create(
    model="MiniCPM-SALA",
    messages=[{"role": "user", "content": "解释什么是深度学习"}],
    max_tokens=1024,
    stream=True,
)

for chunk in stream:
    delta = chunk.choices[0].delta.content
    if delta:
        print(delta, end="", flush=True)
print()
```

## 项目结构

```
SGLang-SALA/
├── README.md                       # 本文档
├── 使用方法.md                      # 使用说明（与 README 内容一致）
├── Dockerfile.public               # 生产镜像构建文件
├── sgl-kernel/                     # SGLang CUDA 内核（多架构重编译用）
│   ├── pyproject.toml
│   ├── CMakeLists.txt
│   ├── csrc/                       # CUDA/C++ 算子源码
│   └── python/sgl_kernel/          # Python 绑定
└── SGLang-MiniCPM-SALA/            # 离线安装包
    ├── install.sh                  # 一键安装脚本（venv + 编译）
    ├── verify.sh                   # 环境验证脚本
    ├── requirements.txt            # 冻结依赖列表
    └── packages/
        ├── sglang-minicpm/         # SGLang MiniCPM 定制版源码
        │   ├── python/             #   - sglang 主包
        │   └── sgl-kernel/         #   - sgl-kernel 内核
        ├── infllmv2_cuda_impl/     # InfLLM v2 稀疏注意力 CUDA 扩展
        └── sparse_kernel/          # Sparse Kernel CUDA 扩展
```

### 核心组件说明

| 组件 | 说明 |
|------|------|
| **SGLang (sglang-minicpm)** | 基于 SGLang 的 MiniCPM 定制版推理框架，包含 HTTP 服务端、调度器、模型加载、KV Cache 管理等 |
| **sgl-kernel** | SGLang 高性能 CUDA 算子库，包含 attention、flash_attn、gemm、MoE、sampling 等算子 |
| **infllm_v2** | InfLLM v2 稀疏注意力实现，包含 block mask、top-k 选择、变长序列注意力等 CUDA 算子 |
| **sparse_kernel** | 稀疏查表内核扩展，支持 SM 80/90/120 架构 |

## 从源码构建镜像

### 使用构建脚本

使用上级目录的 `build_image.sh` 一键构建：

```bash
bash build_image.sh                # 默认使用 1/4 CPU 核心编译
bash build_image.sh --jobs 32      # 手动指定编译并行数
bash build_image.sh --name my:tag  # 自定义镜像名称
```

### 直接使用 Docker 构建

```bash
docker build \
  --file Dockerfile.public \
  --build-arg BUILD_JOBS=16 \
  --build-arg TORCH_CUDA_ARCH_LIST="8.0;8.9;9.0;12.0;12.0a" \
  -t minicpm-sala:v1.0 .
```

### 构建参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `BUILD_JOBS` | `8` | 编译并行数（影响 CMake、Make 和 pip install） |
| `TORCH_CUDA_ARCH_LIST` | `8.0;8.9;9.0;12.0;12.0a` | 目标 GPU 架构列表，按需裁剪可加速编译 |

### 构建过程

1. 基于 CUDA 12.9.1 + Ubuntu 24.04 基础镜像
2. 安装 Python 3.10 和 uv 包管理器
3. 执行 `install.sh`：创建 venv → 编译 sglang → 编译 infllm_v2 → 编译 sparse_kernel → 安装 tilelang 和 flash-linear-attention
4. 从根目录 `sgl-kernel/` 重编译多架构 sgl-kernel（覆盖步骤 3 中的版本）

> **提示：** 构建过程包含大量 CUDA 内核编译，预计耗时 **1~3 小时**，取决于 CPU 核心数和目标架构数量。如果只需支持单一架构（如 A100），可将 `TORCH_CUDA_ARCH_LIST` 设为 `"8.0"` 以大幅缩短编译时间。

## 辅助脚本：run_serve.sh

项目还提供了 `run_serve.sh` 脚本，支持使用评测镜像快速部署纯 API 服务（不含评测逻辑）：

```bash
bash run_serve.sh                                    # 默认参数启动

bash run_serve.sh --model /data/MiniCPM-SALA         # 指定模型路径

```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--gpu` | `2` | GPU 设备号 |
| `--port` | `30000` | 主机映射端口 |
| `--model` | `/home/user/model/MiniCPM-SALA` | 宿主机模型路径 |
| `--image` | `sglang_test_verify:2.2.7` | Docker 镜像 |
| `--input-url` | 无 | 选手提交包 URL（可选，启动前执行 `prepare_env.sh`） |
| `--server-args` | minicpm_flashinfer 默认参数 | SGLang 服务参数 |
| `--tp` | `1` | Tensor Parallel |
| `--max-reqs` | `32` | 最大并发数 |

## 常见问题

### 服务启动后多久可用？

模型加载通常需要 **1~5 分钟**，取决于模型大小和 GPU 性能。可通过轮询 `/health` 接口判断服务是否就绪：

```bash
while ! curl -sf http://localhost:30000/health > /dev/null 2>&1; do
  echo "等待服务就绪..."
  sleep 5
done
echo "服务已就绪！"
```

### 如何查看 GPU 显存占用？

```bash
# 宿主机上执行
nvidia-smi

# 或进入容器
docker exec -it <container_id> nvidia-smi
```

### 端口被占用怎么办？

更换主机映射端口即可，容器内端口通过 `-e PORT` 同步修改：

```bash
docker run --rm --gpus '"device=0"' -p 8080:8080 \
  -v /path/to/model:/models/MiniCPM-SALA \
  -e PORT=8080 \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 如何使用多卡？

通过 `--gpus` 指定多卡，同时设置 `TP_SIZE`：

```bash
docker run --rm --gpus '"device=0,1"' -p 30000:30000 \
  -v /path/to/model:/models/MiniCPM-SALA \
  -e TP_SIZE=2 \
  modelbest-registry.cn-beijing.cr.aliyuncs.com/openbmb/minicpm-sala:v1.0
```

### 显存不足 (OOM) 怎么办？

- **减小 `chunked-prefill-size`**：通过 `SGLANG_SERVER_ARGS` 设置更小的值（如 `4096` 或 `2048`）
- **降低并发数**：设置 `-e MAX_RUNNING_REQUESTS=16` 或更低
- **使用多卡**：增加 `TP_SIZE` 将模型分片到多张 GPU

### 容器启动后立即退出？

```bash
# 查看容器退出日志
docker logs <container_id>

# 常见原因：
# 1. 模型路径错误 → 检查 -v 挂载路径是否正确
# 2. GPU 不可用 → 确认 nvidia-container-toolkit 已安装
# 3. CUDA 版本不兼容 → 确认驱动版本 ≥ 535
```

### 如何查看 Prometheus 指标？

确保 `ENABLE_METRICS=1`（默认开启），然后访问：

```bash
curl http://localhost:30000/metrics
```

可接入 Grafana + Prometheus 进行可视化监控。
